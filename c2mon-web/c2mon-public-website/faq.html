C2MON F.A.Q.


All your favourite questions, with our answers.

Who is behind this?


The software is developed and used by several projects at CERN in Geneva, Switzerland. It is used for monitoring a wide variety of technical infrastructure across the CERN sites.


What makes your product different from other SCADA systems?


It's free. It's multi-platform. It can run as a high-availability cluster. It's scalable to manage large amounts of data. It's easy to write a custom data acquisition module to suit your needs. It's easy to build your own GUI application on top of our API.


Where is the download link?


The software is not yet freely distributed. If you are interested in using C2MON for a monitoring project, or just trying it out, please contact us for a demo package.


How much does it cost?


C2MON is free and based on open source products. Some of these products offer support contracts and enterprise features for a licence cost, if you require these (for cache, database or middleware).


What licence are you using?


The plan is to make the project open source in the near future, so please be patient...


What's inside?


Short answer is Java. For more details on the internals, see below...


Overall architecture


C2MON is a 3-tier SCADA monitoring platform, made up of a data acquisition (DAQ) API, a server layer and a client API. The inter-layer communication is done exclusively through JMS (ActiveMQ). The acquired data is logged in the configured database and kept in-memory in a cache (Ehcache). C2MON comes equipped with a number of available DAQ and client implementations (see below)


Server cluster configuration


The C2MON server can run in a clustered setup, by making use of the Ehcache Terracotta distributed cache technology. This allows to build high-availability services on top of C2MON. The cache can also make use of Terracotta BigMemory ®, for high-performance monitoring requirements of large data sets.


Supported databases


At CERN we run with Oracle, but the platform can run equally well with other standard databases (the demo package even uses the in-memory HSQLDB ).


Available client applications


A number of client applications are already available on top of C2MON (see screenshots). Some of them make use of the IBM ILOG ®, framework, and hence require are licence, but a free GUI is also available.


Available data acquisition modules


Data acquisition modules already exist for acquiring data from a variety of sources, in particular OPC UA, and Siemens/Schneider PLCs. What's more, it is easy to implement new modules to acquire data from your own hardware and protocols.
